## 贡献指南

欢迎为 AI 编程指南做出贡献！本指南将帮助您了解如何参与项目。

### 贡献方式

#### 1. 报告问题

**Bug 报告**
- 使用清晰的问题描述
- 提供复现步骤
- 包含环境信息

**功能建议**
- 描述新功能需求
- 说明使用场景
- 提供参考示例

#### 2. 改进文档

**内容更新**
- 修正错误信息
- 更新过时内容
- 补充缺失信息

**翻译贡献**
- 翻译新内容
- 改进现有翻译
- 校对文档质量

#### 3. 代码贡献

**新功能开发**
- 实现新功能
- 添加测试用例
- 更新文档

**Bug 修复**
- 修复已知问题
- 添加回归测试
- 更新相关文档

### 贡献流程

#### 1. Fork 项目

```bash
# Fork 项目到您的 GitHub 账户
# 然后克隆到本地
git clone https://github.com/your-username/ai-guide.git
cd ai-guide
```

#### 2. 创建分支

```bash
# 创建功能分支
git checkout -b feature/your-feature-name

# 或修复 Bug
git checkout -b fix/bug-description
```

#### 3. 进行更改

**文档编辑**
- 使用 Markdown 格式
- 遵循现有风格
- 添加必要的元数据

**代码修改**
- 遵循代码规范
- 添加注释
- 编写测试

#### 4. 提交更改

```bash
# 添加更改
git add .

# 提交（使用清晰的提交信息）
git commit -m "Add: 添加新功能描述"

# 推送到您的 Fork
git push origin feature/your-feature-name
```

**提交信息规范**
- `Add:` 新功能
- `Fix:` Bug 修复
- `Update:` 内容更新
- `Docs:` 文档改进
- `Refactor:` 代码重构

#### 5. 创建 Pull Request

1. 在 GitHub 上打开您的 Fork
2. 点击 "New Pull Request"
3. 填写 PR 描述
4. 等待审查

**PR 描述模板**
```markdown
## 更改类型
- [ ] 新功能
- [ ] Bug 修复
- [ ] 文档更新
- [ ] 其他

## 描述
简要描述更改内容

## 相关 Issue
关联的 Issue 编号（如有）

## 测试
描述如何测试这些更改
```

### 内容贡献指南

#### 文档结构

**Front Matter（元数据）**
```yaml
---
title: 页面标题
weight: 1  # 排序权重
---
```

**内容格式**
- 使用 Markdown 语法
- 标题层级清晰
- 代码块添加语言标识
- 链接使用相对路径

#### 写作规范

**语言风格**
- 使用简体中文
- 专业但易懂
- 避免过于口语化
- 保持一致性

**技术术语**
- 首次出现时提供英文
- 使用标准术语
- 保持术语一致性

**代码示例**
- 提供完整可运行的示例
- 添加必要的注释
- 说明预期结果

### 代码贡献指南

#### 代码规范

**Python**
- 遵循 PEP 8
- 使用类型提示
- 添加文档字符串

**JavaScript/TypeScript**
- 使用 ESLint
- 遵循 Airbnb 风格
- 添加 JSDoc 注释

#### 测试要求

- 新功能必须包含测试
- 测试覆盖率不低于 80%
- 所有测试必须通过

### 审查流程

#### PR 审查标准

1. **内容质量**
   - 信息准确
   - 格式正确
   - 易于理解

2. **代码质量**
   - 符合规范
   - 性能良好
   - 安全性考虑

3. **完整性**
   - 包含必要文档
   - 测试覆盖充分
   - 无破坏性更改

#### 审查反馈

- 审查者会提供反馈
- 根据反馈进行修改
- 重新提交更改

### 社区行为准则

#### 我们的承诺

- **友好和包容**：欢迎所有背景的贡献者
- **尊重差异**：尊重不同的观点和经验
- **建设性反馈**：提供有帮助的反馈
- **专业行为**：保持专业和礼貌

#### 不可接受的行为

- 骚扰性语言或图像
- 人身攻击
- 发布他人隐私信息
- 其他不当行为

### 获取帮助

#### 常见问题

**如何开始贡献？**
- 查看 "Good First Issue" 标签
- 阅读本文档
- 在 Discussions 中提问

**如何选择贡献内容？**
- 查看 Issue 列表
- 选择感兴趣的主题
- 从小改动开始

**遇到问题怎么办？**
- 在 Discussions 中提问
- 查看现有 Issue
- 联系维护者

#### 联系方式

- **GitHub Issues**：报告问题
- **GitHub Discussions**：讨论和提问
- **Email**：项目维护者邮箱（如有）

### 贡献者认可

#### 贡献者列表

所有贡献者都会在以下位置被认可：
- README.md 贡献者列表
- 项目网站贡献者页面
- 发布说明中提及

#### 贡献类型

- **代码贡献**：功能开发、Bug 修复
- **文档贡献**：内容编写、翻译
- **社区贡献**：回答问题、帮助他人
- **设计贡献**：UI/UX 设计、视觉资源
