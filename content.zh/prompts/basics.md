## 提示词基础

提示词（Prompt）是与 AI 模型交互的核心方式。掌握提示词编写技巧是有效使用 AI 编程工具的基础。

### 什么是提示词

提示词是您向 AI 模型提供的输入文本，用于指导模型生成期望的输出。好的提示词能够：
- 明确任务目标
- 提供必要上下文
- 指定输出格式
- 控制输出质量

### 提示词的基本结构

#### 1. 角色定义

**示例**：
```
你是一位经验丰富的 Python 开发工程师，擅长编写清晰、高效的代码。
```

**作用**：
- 设定 AI 的角色和专业领域
- 影响 AI 的回答风格和专业程度

#### 2. 任务描述

**示例**：
```
请帮我编写一个函数，用于计算两个日期之间的工作日数量，排除周末和指定节假日。
```

**要点**：
- 清晰明确地描述任务
- 包含必要的细节和要求

#### 3. 上下文信息

**示例**：
```
项目使用 Python 3.9+，遵循 PEP 8 代码规范，使用 type hints。
现有代码结构如下：
[提供相关代码]
```

**作用**：
- 帮助 AI 理解项目背景
- 确保输出符合项目要求

#### 4. 输出要求

**示例**：
```
请提供：
1. 完整的函数代码
2. 单元测试用例
3. 使用示例
```

**要点**：
- 明确指定输出格式
- 列出所需内容

### 提示词编写原则

#### 1. 清晰明确

**好的提示词**：
```
创建一个 REST API 端点，接收用户 ID，返回该用户的订单列表。
使用 FastAPI 框架，包含错误处理和日志记录。
```

**不好的提示词**：
```
做个 API
```

#### 2. 提供上下文

**好的提示词**：
```
在以下代码基础上，添加用户认证功能：
[提供现有代码]

要求：
- 使用 JWT token
- 支持登录和注册
- 密码加密存储
```

**不好的提示词**：
```
加个登录功能
```

#### 3. 分步骤说明

**示例**：
```
请按以下步骤重构这个函数：
1. 提取重复逻辑到独立函数
2. 添加类型提示
3. 改进错误处理
4. 添加文档字符串

原函数：
[提供代码]
```

#### 4. 使用示例

**示例**：
```
编写一个函数，将字符串转换为驼峰命名。

示例：
- "hello world" -> "helloWorld"
- "user name" -> "userName"
- "MY VARIABLE" -> "myVariable"
```

### 常见提示词模式

#### 1. 代码生成

**模式**：
```
作为 [角色]，请 [任务描述]。

要求：
- [要求1]
- [要求2]
- [要求3]

示例：
[提供示例]
```

**实际应用**：
```
作为 Python 开发工程师，请编写一个数据验证函数。

要求：
- 验证邮箱格式
- 验证手机号格式
- 返回验证结果和错误信息

示例：
validate("user@example.com", "13800138000")
# 返回: {"email": True, "phone": True, "errors": []}
```

#### 2. 代码审查

**模式**：
```
请审查以下代码，检查：
1. 潜在 Bug
2. 性能问题
3. 安全问题
4. 代码风格

代码：
[提供代码]
```

#### 3. 代码重构

**模式**：
```
请重构以下代码，使其：
- 更易读
- 更易维护
- 性能更好

原代码：
[提供代码]
```

#### 4. 问题解答

**模式**：
```
请解释以下代码的功能和工作原理：
[提供代码]

并回答：
1. 这段代码做了什么？
2. 为什么这样实现？
3. 有什么可以改进的地方？
```

### 提示词优化技巧

#### 1. 使用关键词

**有效关键词**：
- "请"、"请帮我"：礼貌且明确
- "详细"、"完整"：要求更详细的输出
- "简洁"、"简要"：要求简洁的输出
- "示例"、"举例"：要求提供示例

#### 2. 结构化输出

**使用列表**：
```
请提供：
1. 功能说明
2. 代码实现
3. 测试用例
4. 使用示例
```

**使用表格**：
```
请以表格形式列出：
| 功能 | 实现方式 | 优点 | 缺点 |
```

#### 3. 迭代优化

**第一次提示**：
```
写个排序函数
```

**优化后**：
```
作为 Python 开发工程师，请编写一个通用的排序函数。

要求：
- 支持升序和降序
- 支持自定义比较函数
- 处理边界情况（空列表、单个元素）
- 包含类型提示和文档字符串
- 时间复杂度 O(n log n)

请提供：
1. 函数实现
2. 单元测试
3. 使用示例
```

### 常见错误

#### 1. 提示词过于简单

**错误**：
```
写代码
```

**改进**：
```
编写一个 Python 函数，实现快速排序算法，包含类型提示和文档字符串。
```

#### 2. 缺少上下文

**错误**：
```
优化这个
```

**改进**：
```
优化以下函数的性能，当前时间复杂度为 O(n²)，目标是 O(n log n)：
[提供代码]
```

#### 3. 要求不明确

**错误**：
```
做个好点的版本
```

**改进**：
```
重构以下代码，使其：
- 遵循 SOLID 原则
- 提高可测试性
- 添加错误处理
- 改进代码可读性
```

### 实践练习

#### 练习 1：代码生成

**任务**：生成一个用户管理类

**提示词模板**：
```
作为 Python 开发工程师，请创建一个 UserManager 类。

功能要求：
- 用户注册（用户名、邮箱、密码）
- 用户登录验证
- 获取用户信息
- 更新用户信息
- 删除用户

技术要求：
- 使用 SQLAlchemy ORM
- 密码使用 bcrypt 加密
- 包含输入验证
- 添加异常处理
- 包含完整的类型提示

请提供：
1. 类定义和实现
2. 数据库模型定义
3. 单元测试
```

#### 练习 2：代码审查

**任务**：审查代码质量

**提示词模板**：
```
请审查以下代码，检查：
1. 潜在 Bug 和安全漏洞
2. 性能问题
3. 代码风格和最佳实践
4. 可维护性问题

代码：
[提供代码]

请以以下格式输出：
- Bug：[列出发现的 Bug]
- 安全问题：[列出安全问题]
- 性能问题：[列出性能问题]
- 改进建议：[提供改进建议]
```
